CLI commands

Complete reference for every orbit subcommand, including flags, defaults, and exit behavior.

orbit init #

Bootstrap a new project by creating an orbit.yml file in the current directory. Use --template to pick a starter.

$ orbit init --template node --name my-app
✓ Created orbit.yml
✓ Linked to project "my-app"

orbit deploy #

Build the project and push to the target environment. Supports incremental builds and rollback.

FlagDefaultDescription
--envstagingTarget environment name.
--no-cachefalseDisable layer caching for this deploy.
--detachfalseReturn immediately after queuing.
$ orbit deploy --env production
# building → uploading → switching traffic
✓ Released v138 in 14.2s

orbit env #

Manage environment variables and secrets. Values are encrypted client-side before transmission.

$ orbit env set DATABASE_URL="postgres://…"
$ orbit env list
NAME           VALUE
DATABASE_URL   (encrypted)
NODE_ENV       production

orbit logs #

Stream logs from running services. Use --follow to tail and --since to filter by time.

$ orbit logs --follow --since 10m
[14:02:11] web.1  GET /api/users 200 18ms
[14:02:12] web.1  GET /api/posts 200 22ms
