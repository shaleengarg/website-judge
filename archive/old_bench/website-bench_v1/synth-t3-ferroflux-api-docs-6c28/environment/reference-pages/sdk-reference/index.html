SDK Reference

First-party FerroFlux SDKs wrap the REST and streaming APIs in idiomatic clients. Choose your language below, install the package, and start ingesting sensor data in under a minute.

Official Clients

Py Python SDK

ferroflux-py · 2.4.1 · MIT

Install

bash# requires Python ≥ 3.9
pip install ferroflux

Usage

pythonfrom ferroflux import Client

client = Client(api_key="ff_live_…")

stream = client.streams.create(
    name="reactor-3-thermal",
    rate_hz=250,
)

for sample in stream.subscribe():
    print(sample.t, sample.value)

JS Node.js SDK

@ferroflux/node · 2.4.0 · MIT

Install

bash# requires Node ≥ 18
npm install @ferroflux/node

Usage

javascriptimport { Client } from "@ferroflux/node";

const client = new Client({
  apiKey: process.env.FF_KEY,
});

const stream = await client.streams.create({
  name: "reactor-3-thermal",
  rateHz: 250,
});

for await (const s of stream.subscribe()) {
  console.log(s.t, s.value);
}
Note: Both SDKs share the same wire protocol and authentication scheme. Tokens are interchangeable between clients within the same workspace.
