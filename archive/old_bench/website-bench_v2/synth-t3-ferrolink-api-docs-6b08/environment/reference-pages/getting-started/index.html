Getting Started with FerroLink

FerroLink streams sensor telemetry from industrial equipment — pumps, turbines, PLC controllers, fleet edge gateways — directly into your cloud data pipelines. This guide walks you through provisioning credentials, installing the SDK, and issuing your first authenticated request in under five minutes.

Overview

The FerroLink platform exposes a uniform REST surface over three resource groups: streams, devices, and alerts. Time-series payloads ingested at the edge are batched, signed with HMAC-SHA256, and forwarded to a regional ingest endpoint where they are exposed for downstream consumption via WebSocket or pull-based polling.

All endpoints are versioned under the path prefix /v3/ and require a bearer token issued against your organization's API key. Responses are JSON-encoded and conform to the schemas documented in Data Models.

Tip. If you are migrating from FerroLink v2.x, see the changelog for the list of breaking changes — most notably, telemetry payloads now use epoch_ns rather than ISO-8601 strings.

Quickstart

  1. 1
    Obtain an API key

    Sign in to the FerroLink Console, navigate to Settings → API Keys, and click Generate key. Scope the key to streams:read for this tutorial. Keys are shown once — store yours in a secrets manager.

  2. 2
    Install the SDK npm

    The official Node.js client wraps authentication, pagination, and retry logic.

    # add ferrolink-sdk to your project
$ npm install ferrolink-sdk@^3.4 --save

# verify the install
$ npx ferrolink --version
ferrolink-sdk 3.4.1  (node 20.11.0, openssl 3.2.0)
  3. 3
    Make your first request

    The following snippet authenticates with your key and lists the first page of streams available to your organization. Replace FL_API_KEY with the token you generated in step 1. 

    import { FerroLink } from 'ferrolink-sdk';

const client = new FerroLink({
  apiKey: process.env.FL_API_KEY,
  region: 'eu-west-1',
});

const res = await fetch('https://api.ferrolink.io/v3/streams?limit=10', {
  headers: {
    'Authorization': `Bearer ${client.token}`,
    'Accept':        'application/json',
  },
});

const { data } = await res.json();
console.log(`fetched ${data.length} streams`);
// → fetched 10 streams

    A successful response returns HTTP 200 OK with a data[] array of StreamObject entries and a next_cursor field for pagination.

Prerequisites

Confirm your environment meets the minimum supported versions below. Older runtimes may negotiate deprecated TLS ciphers and be rejected by the ingest gateway.

Requirement Minimum Version
Node.js runtime (for ferrolink-sdk) 18.17.0   LTS recommended: 20.x
OpenSSL (TLS 1.3, X25519 keyshare) 3.0.8
FerroLink gateway firmware (edge appliance) fl-gw 2.11.4
