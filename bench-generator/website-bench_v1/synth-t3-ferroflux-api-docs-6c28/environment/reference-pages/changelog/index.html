Release Notes

Changelog

A chronological record of changes, additions, and fixes across FerroFlux platform releases. Subscribe to the RSS feed or watch the GitHub repo to get notified of new releases.

v2.4.0

Released March 14, 2025 Latest
  • AddedNew /streams/{id}/aggregates endpoint for rolling window statistics (min, max, mean, p95).
  • AddedWebSocket subscription filters via where query expressions on tag values.
  • AddedOfficial Rust SDK ferroflux-rs v0.1 — early access via crates.io.
  • ChangedDefault pagination limit raised from 100 to 250 items per page on list endpoints.
  • FixedRace condition in batched ingest causing dropped samples under sustained >50k msg/s loads.

v2.3.0

Released January 22, 2025
  • AddedRegion-aware ingest endpoints (eu-west, ap-south) with automatic failover.
  • AddedAPI key scoping: read-only, write-only, and per-stream permissions in the dashboard.
  • ChangedIncreased maximum payload size from 1 MB to 4 MB on POST /streams/{id}/data.
  • ChangedDeprecated the ?expand=metadata query parameter — fields are now returned by default.
  • FixedTimestamp parsing accepts RFC 3339 fractional seconds beyond microsecond precision.
  • FixedNode.js SDK no longer leaks sockets when reconnecting after long idle periods.

v2.2.0

Released November 5, 2024
  • AddedServer-Sent Events (SSE) transport as an alternative to WebSockets for HTTP/1.1 clients.
  • AddedOpenAPI 3.1 spec published at /openapi.json with full schema definitions.
  • ChangedRate-limit headers renamed to standard RateLimit-* per IETF draft 08.
  • FixedSchema validation incorrectly rejected nested arrays with mixed numeric types.
  • FixedDashboard timezone selector reset on every page navigation.

v2.1.0

Released August 19, 2024
  • AddedPython SDK 2.1 with native asyncio support and typed response models.
  • AddedStream tagging API — attach up to 32 key/value tags per stream for filtering.
  • ChangedAuthentication tokens now use a fflx_ prefix for easier secret-scanning.
  • FixedCORS preflight responses missing Access-Control-Allow-Headers on signed requests.

v2.0.0

Released May 6, 2024 Major
  • AddedBrand new v2 REST API surface with cursor-based pagination and consistent error envelope.
  • AddedReal-time WebSocket gateway at wss://api.ferroflux.io/v2/live with sub-50ms latency.
  • AddedMulti-tenant workspaces and team member invitations.
  • ChangedBreaking: v1 endpoints under /api/ moved to /v1/ and scheduled for sunset on 2025-05-06.
  • ChangedBreaking: response envelope now always wraps payload in a data field.
  • FixedClock-skew tolerance on signed requests extended to ±300 seconds.

v1.9.0

Released February 11, 2024 Legacy
  • AddedBulk delete operations on streams matching tag selectors.
  • AddedAudit log endpoint for security and compliance review.
  • ChangedDefault data retention raised from 14 to 30 days on the Pro tier.
  • FixedMemory leak in the Go SDK long-polling client under high reconnect rates.
  • FixedInconsistent ordering of samples returned with ?sort=desc when timestamps tied.
