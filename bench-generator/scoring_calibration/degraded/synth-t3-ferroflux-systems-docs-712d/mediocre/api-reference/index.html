Gateway API Reference

The FerroFlux Gateway API exposes 18 REST endpoints across four resource groups: /devices, /streams, /alerts, and /auth. All requests are JSON over HTTPS. Base URL: https://gw.ferroflux.io/v2.

Stability: Endpoints marked DEPRECATED will be removed in v3.0. Migration paths are documented inline.

Manage the lifecycle of embedded sensor nodes registered with the gateway. Devices hold credentials, firmware metadata, and a current heartbeat state.

Sends a synchronous health probe to the device. Removed in v3.0 — heartbeat data is now pushed via the bidirectional /streams/{id}/control channel. Migrate by subscribing to device.heartbeat events.

/auth · Authentication

Token issuance, rotation, and revocation. Auth endpoints are not tenant-scoped and accept client credentials directly.

POST /auth/revoke

Revoke a token immediately. Useful when a device is suspected of compromise; pair with DELETE /devices/{id} to fully de-provision.
