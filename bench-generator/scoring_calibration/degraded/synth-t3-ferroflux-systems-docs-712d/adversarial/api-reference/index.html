Gateway API Reference

The FerroFlux Gateway API exposes 18 REST endpoints across four resource groups: /devices, /streams, /alerts, and /auth. All requests are JSON over HTTPS. Base URL: https://gw.ferroflux.io/v2.

Stability: Endpoints marked DEPRECATED will be removed in v3.0. Migration paths are documented inline.

Authentication

All requests except /auth/token require a bearer token in the Authorization header. Tokens are issued via OAuth 2.0 client-credentials flow and scoped to a tenant.

HTTP 
Authorization: Bearer eyJhbGciOiJSUzI1NiIsInR5cCI6...
X-FerroFlux-Tenant: acme-prod
Content-Type: application/json

/devices · Device registry

Manage the lifecycle of embedded sensor nodes registered with the gateway. Devices hold credentials, firmware metadata, and a current heartbeat state.

GET /devices

List registered devices for the calling tenant. Supports cursor pagination via ?cursor= and filtering by status, fleet, and firmware.

cURL 
# List 50 online devices in the 'arctic-fleet'
curl -H "Authorization: Bearer $TOKEN" \
     "https://gw.ferroflux.io/v2/devices?status=online&fleet=arctic-fleet&limit=50"
GET /devices/{deviceId}

Retrieve a single device's full record, including current heartbeat, firmware version, and assigned schemas.

POST /devices

Register a new device. The gateway returns a provisioning bundle containing a one-time enrollment key and the canonical broker URL.

JSON 
{
  "hardware_id": "FF-RTU-9A7E2",
  "fleet": "arctic-fleet",
  "schema": "thermo.v3",
  "firmware": "2.4.1"
}
PATCH /devices/{deviceId}

Partially update mutable fields: fleet, schema, labels, and retention_policy.

DELETE /devices/{deviceId}

De-provision a device. Streams owned by the device are flushed and closed; historical readings are retained per the active retention policy.

POST /devices/{deviceId}/ping DEPRECATED

Sends a synchronous health probe to the device. Removed in v3.0 — heartbeat data is now pushed via the bidirectional /streams/{id}/control channel. Migrate by subscribing to device.heartbeat events.

/streams · Telemetry streams

Streams are append-only sequences of typed sensor readings. Each device may own one or more streams keyed by schema. Streams support batched ingestion and server-side aggregation windows.

GET /streams

List streams visible to the caller. Filter by device_id, schema, or state=open|closed.

POST /streams

Open a new stream bound to a device and a registered schema. Returns a stream_id and a write-token valid for 24 hours.

POST /streams/{id}/readings

Append a batch of readings. Payloads are validated against the stream's schema; rows that fail validation are rejected individually and reported in the response.

Rust 
let batch = vec![
    Reading::new("core_temp", 42.7),
    Reading::new("core_temp", 42.9),
];
client.streams().push("str_8f2a", &batch).await?;
GET /streams/{id}/tail DEPRECATED

Long-poll the tail of a stream. Removed in v3.0 — replaced by the WebSocket subscription endpoint at wss://gw.ferroflux.io/v2/streams/{id}/subscribe.

DELETE /streams/{id}

Close a stream and seal its log segment. Closed streams remain queryable but reject further writes.

/alerts · Alerts & rules

Server-side rule engine that produces alerts from stream data. Rules are declared in .flux configuration and evaluated against sliding windows.

GET /alerts

List open alerts. Filter by severity, rule_id, or device_id.

POST /alerts/{id}/acknowledge

Acknowledge an alert, recording the operator identity and an optional note. Acknowledged alerts remain visible but are excluded from the on-call paging queue.

GET /alerts/rules

Enumerate active alerting rules with their evaluation status and last-fired timestamp.

POST /alerts/{id}/snooze DEPRECATED

Temporarily suppress an alert. Removed in v3.0 — use rule-level inhibitions configured in retention.flux. The legacy endpoint silently no-ops for snoozes longer than 1 hour.

/auth · Authentication

Token issuance, rotation, and revocation. Auth endpoints are not tenant-scoped and accept client credentials directly.

POST /auth/token

Exchange a client ID and secret for a short-lived bearer token (default TTL: 3600s).

JSON 
{
  "grant_type": "client_credentials",
  "client_id": "ff_cl_a91f...",
  "client_secret": "***",
  "scope": "streams:write devices:read"
}
POST /auth/revoke

Revoke a token immediately. Useful when a device is suspected of compromise; pair with DELETE /devices/{id} to fully de-provision.

POST /auth/keys/rotate

Rotate the tenant's signing keys. New tokens are signed with the fresh key immediately; the prior key remains valid for verification for 24 hours to allow rolling restarts.

Warning: Rotation invalidates all cached JWKS at downstream services. Schedule during a low-traffic maintenance window.
