Getting Started

A five-step walkthrough to install the FerroFlux SDK v2.4, provision your first MQTT broker profile, register an embedded device, push a sensor reading, and verify it lands in the operator dashboard.

Prerequisites  ·  Rust toolchain 1.74+, a reachable FerroFlux Gateway (self-hosted or cloud), and a valid FFX_API_TOKEN exported in your shell.
1

Install the SDK & CLI

Install the ferroflux command-line tool and the Rust SDK from crates.io. The CLI is statically linked and ships with the embedded gateway shim used in steps 3–5.

bash 
# install the CLI globally
cargo install ferroflux-cli --version "2.4.0"

# add the SDK to your project
cargo add ferroflux-sdk@2.4 --features "mqtt,tls"
2

Configure a Broker Profile

Profiles are declared in ferroflux.flux and define how the SDK reaches your MQTT broker. The generator scaffolds a sane default for TLS on port 8883.

flux 
# ferroflux.flux
broker "primary" {
  host         = "mqtts://broker.ferroflux.io"
  port         = 8883
  tls          = true
  client_id    = "edge-${HOSTNAME}"
  keepalive_s  = 30
  retention    = "7d"
}
3

Register a Device

Each physical sensor is registered against a schema. The CLI returns a device handle (dev_…) and a one-time provisioning secret. Store the secret in your secure enclave or HSM.

bash 
ferroflux devices register \
  --name      "floor-3-temp-01" \
  --schema    "thermal/v2" \
  --broker    "primary" \
  --tags      "site=oslo,line=A"

# → dev_8f3a21c0  (secret: ffx_sk_••••••••••••)
4

Push Your First Reading

Open a stream from your Rust application and emit a typed sample. The SDK batches and compresses readings transparently; flush().await guarantees durability before continuing.

rust 
use ferroflux_sdk::{Client, Reading, schema::Thermal};

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let client = Client::from_env().await?;
    let mut stream = client.stream("dev_8f3a21c0").await?;

    stream.push(Reading::<Thermal> {
        celsius:  21.4,
        humidity: 0.38,
        ts:       chrono::Utc::now(),
    }).await?;

    stream.flush().await?;
    Ok(())
}
5

Verify in the Dashboard

The CLI's tail subcommand streams live readings as they ingress the gateway. If you see your sample echoed within 2 seconds, you're done — the same data is now visible in the operator dashboard under Devices → floor-3-temp-01.

bash 
ferroflux streams tail dev_8f3a21c0 --since 1m

# 14:22:08.114  thermal/v2  celsius=21.4  humidity=0.38  ✓ ingested

Note DEPRECATED
The legacy ferroflux verify --device flow used in v1.x has been removed; use streams tail instead.
