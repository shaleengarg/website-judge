Changelog

v2.4.1 Released August 12, 2024 stable
Added
  • New ferroflux::stream::Backpressure policy with adaptive window sizing for embedded targets.
  • Configuration DSL now supports inline @import directives for sharing broker profiles across deployments.
  • Gateway endpoint POST /v1/devices/:id/heartbeat for low-power keepalive on cellular links.
  • Rust SDK: async fn Device::flush_with_timeout(...) returning structured FlushReport.
Changed
  • Default MQTT QoS lowered from 2 to 1 for new device registrations. Existing devices unaffected.
  • TLS handshake timeout raised to 12s to accommodate constrained NB-IoT modems.
  • CLI fluxctl now prints color-coded severity in --watch mode.
Fixed
  • Race condition when a stream is closed during tick() on single-threaded async executors.
  • Incorrect schema fingerprint for nested Map<String, Bytes> fields.
  • Panic in RetentionPolicy::compact() when storage backend reports a future timestamp.
Deprecated
  • DEPRECATED Device::push_raw(&[u8]) — use Device::push<T: Encode>(value). Removal planned for v2.6.
  • DEPRECATED Endpoint GET /v1/alerts/by-tag — superseded by query parameter ?tag= on /v1/alerts.
v2.3.0 Released May 28, 2024 stable
Added
  • WebSocket transport (wss://) as a first-class alternative to MQTT for environments behind restrictive proxies.
  • Configuration key broker.fallback_endpoints accepting an ordered list of failover URLs.
  • Alert::correlation_id field, propagated through downstream webhooks.
flux 
# Excerpt: new fallback configuration
broker "north-eu" {
  endpoint          = "mqtts://north-eu.ferroflux.io:8883"
  fallback_endpoints = [
    "mqtts://north-eu-backup.ferroflux.io:8883",
    "wss://relay.ferroflux.io:443/mqtt",
  ]
  reconnect_after   = 5s
}
Changed
  • Telemetry batch size raised from 32 to 64 by default; tunable via stream.batch_size.
  • The fluxctl deploy command now requires explicit --profile when multiple profiles are detected.
Fixed
  • Memory leak in long-lived StreamSubscriber when consumer drops mid-batch.
  • Schema migration silently dropped Option<T> fields with default None.
  • JWT validation rejected tokens issued exactly at the nbf boundary.
Deprecated
  • DEPRECATED Environment variable FFX_LEGACY_AUTH=1 — basic auth fallback will be removed in v3.0.
v2.2.4 Released February 14, 2024 LTS
Added
  • Long-Term Support designation — v2.2.x receives security patches through Q4 2025.
  • Hardware-attested device identities on supported MCUs (ESP32-S3, STM32H7, nRF9160).
  • Prometheus exporter on port 9477 exposing 28 gateway-internal metrics.
  • New retention strategy tiered_cold for cost-tiered S3-compatible storage.
Changed
  • Default log format switched from text to JSON; revert with logging.format = "text".
  • SDK MSRV bumped to Rust 1.74.
Security
  • CVE-2024-2114 — patched a denial-of-service vector in the schema parser when handling deeply nested types. Severity: Medium.
Fixed
  • Reconnect storm when broker certificate rotated mid-session.
  • Off-by-one in batch ordering under specific clock-skew conditions.
v2.1.0 Released November 3, 2023 stable
Added
  • Alert routing rules with declarative match blocks in .flux.
  • no_std feature flag for the SDK — verified on Cortex-M4F targets.
  • Per-stream encryption-at-rest keys.
Changed
  • Auth token TTL default reduced from 24h to 6h; refresh tokens unchanged.
Fixed
  • Spurious ErrSchemaMismatch when a non-required field is added in a minor schema bump.
  • Gateway returned 500 instead of 413 for oversized payloads.
Deprecated
  • DEPRECATED stream.legacy_framing — the v1 framing protocol will be removed in v2.5.
v2.0.0 Released July 18, 2023 major

FerroFlux 2.0 is a full rewrite of the gateway and SDK in async Rust. The v1.x C/C++ daemon is retained only as a compatibility shim and is scheduled for end-of-life in December 2024.

Added
  • Brand-new Rust SDK with async/await, zero-copy serialization via rkyv, and structured errors.
  • Declarative .flux configuration DSL replacing the verbose INI files of v1.x.
  • Multi-tenant project isolation at the gateway level.
  • Native OpenTelemetry traces emitted from all SDK call sites.
Changed
  • Wire protocol now uses CBOR + Zstd framing. Legacy framing remains available behind a feature flag.
  • Authentication switched to OAuth 2.1 with PKCE for interactive clients and mutual TLS for devices.
Removed
  • The ferroflux-c static library — superseded by the Rust SDK's C FFI bindings.
  • The HTTP/1.0-only fallback transport.
Deprecated
  • DEPRECATED The entire v1.x configuration format. A migration tool ships as fluxctl migrate-v1.
