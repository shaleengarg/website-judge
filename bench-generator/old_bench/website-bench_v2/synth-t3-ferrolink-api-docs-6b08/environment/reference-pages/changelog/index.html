Changelog

Release history for the FerroLink REST API, gateway firmware, and SDK packages. All notable changes since v1.0.0 are documented here in reverse chronological order.

Recent Releases

v3.4.1 2024-11-04 Patch · Stream backpressure fixes
  • FIXEDResolved race condition in POST /streams/:id/ingest when concurrent producers exceed 4096 msg/s, which caused duplicated TelemetryPoint entries.
  • FIXEDGateway firmware 3.4.1-gw now correctly honors the retain_on_disconnect flag for MQTT bridged streams.
  • CHANGEDIncreased default WebSocket idle timeout from 30s to 90s to reduce reconnect churn on cellular links.
v3.4.0 2024-10-12 Minor · Alert pipelines & webhook v2
  • ADDEDNew endpoint POST /alerts/pipelines for composing multi-stage alert evaluation chains with per-stage debounce windows.
  • ADDEDWebhookEvent.v2 envelope, including delivery_attempt, signature_alg, and HMAC-SHA256 signed payloads.
  • CHANGEDAlertPayload.severity now accepts info in addition to warn|critical|fatal.
  • DEPRECATEDWebhookEvent.v1 envelope. Will be removed in v4.0.0; migration guide forthcoming.
  • FIXEDPagination cursors no longer expire prematurely when since & until overlap UTC day boundary.

v3.3.0 2024-08-20 Minor · Device fleet APIs
  • ADDEDBulk endpoint POST /devices/batch accepts up to 500 DeviceRecord entries per request with atomic rollback.
  • ADDEDdevice.firmware_channel field with values stable|beta|edge for staged rollout control.
  • CHANGEDGET /devices now returns last_seen_at as RFC3339; the legacy last_seen epoch field remains for backwards compatibility until v4.
  • FIXEDAuth tokens issued via POST /auth/refresh were not honoring the scopes parameter when downscoping. Now correctly intersects requested with parent scopes.
v3.2.0 2024-06-03 Minor · Query language & aggregations
  • ADDEDFerroQL — a SQL-subset query language for POST /streams/:id/query. Supports SELECT, WHERE, GROUP BY tumble(…), and HAVING.
  • ADDEDAggregations: p50, p95, p99, stddev, rate(…).
  • DEPRECATEDGET /streams/:id/window?op=mean&… legacy aggregator. Use FerroQL.

Historical Releases

v3.0.0 2024-02-14 Major · Unified gateway protocol
  • ADDEDUnified gRPC + REST gateway protocol. SDKs auto-negotiate transport via ALPN.
  • CHANGEDBase URL moved from api.ferrolink.io/v2 to api.ferrolink.io/v3.
  • CHANGEDAll resource IDs are now ULIDs (was: UUIDv4).
  • DEPRECATEDThe /legacy/* namespace will be sunset 2025-02-14.
v2.1.0 2023-09-18 Minor · Multi-region routing
  • ADDEDRegion-pinned streams (region: eu-west-1 | us-east-1 | ap-southeast-2).
  • FIXEDEdge cache invalidation lag on DELETE /streams/:id.
v1.0.0 2022-11-01 Initial public release
  • ADDEDCore endpoints: /streams, /devices, /alerts.
  • ADDEDJSON-only ingest pipeline at up to 1024 msg/s per stream.
  • ADDEDOAuth 2.0 client-credentials flow and PAT-style API keys.
