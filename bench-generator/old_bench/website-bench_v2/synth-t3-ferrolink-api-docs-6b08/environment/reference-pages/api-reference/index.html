API Reference

FerroLink exposes 14 REST endpoints across three resource groups for ingesting, retrieving, and reacting to industrial sensor telemetry. All endpoints accept and return application/json and require a bearer token.

AUTH ▸ All requests must include Authorization: Bearer <FERROLINK_API_KEY>. Tokens are scoped per-organization and rotate every 90 days. See Getting Started.

Streams 5 endpoints · /v3/streams

GET /v3/streams
List telemetry streams

Returns a paginated collection of StreamObject records visible to the authenticated organization. Streams can be filtered by device_id, status, or creation window.

Parameter
Type
Description
limit
integer
Page size, 1–500. Default 50.
cursor
string
Opaque pagination token from previous response.
status
enum
One of active, paused, error.
cURL
Headers
Response
Request
# list active streams
curl -X GET "https://api.ferrolink.io/v3/streams?status=active&limit=50" \
  -H "Authorization: Bearer $FL_KEY" \
  -H "Accept: application/json"
200 OK · application/json
{
  "data": [
    {
      "id": "strm_8H2kQa",
      "device_id": "dev_TX-441",
      "status": "active",
      "rate_hz": 240,
      "created_at": "2024-11-03T14:02:11Z"
    }
  ],
  "cursor": "eyJvIjo1MH0"
}
POST /v3/streams
Create a stream

Provisions a new ingestion stream bound to one or more devices. The request body must satisfy the StreamObject schema; device_id and rate_hz are required.

Field
Type
Description
device_id *
string
Identifier of a registered DeviceRecord.
rate_hz *
integer
Sample rate, 1–10000 Hz.
encoding
enum
cbor (default) or json.
retention_days
integer
1–365. Defaults to plan minimum.
cURL
Headers
Response
Request
curl -X POST "https://api.ferrolink.io/v3/streams" \
  -H "Authorization: Bearer $FL_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "device_id": "dev_TX-441",
    "rate_hz": 240,
    "encoding": "cbor",
    "retention_days": 30
  }'
201 Created
{
  "id": "strm_R9pLm2",
  "status": "provisioning",
  "ingest_url": "wss://ingest.ferrolink.io/s/R9pLm2",
  "created_at": "2024-11-22T09:14:00Z"
}
DELETE /v3/streams/{id}
Delete a stream

Permanently terminates ingestion and purges buffered telemetry older than the stream's retention window. This call is idempotent — a second invocation returns 204 No Content. Associated AlertPayload rules are not removed.

Parameter
Type
Description
id *
path
Stream identifier, e.g. strm_R9pLm2.
purge
boolean
If true, deletes all historical points regardless of retention.
cURL
Headers
Response
Request
curl -X DELETE "https://api.ferrolink.io/v3/streams/strm_R9pLm2?purge=true" \
  -H "Authorization: Bearer $FL_KEY"
204 No Content
# empty response body
# headers:
#   X-FerroLink-Request-Id: 7c1f...e22
#   X-RateLimit-Remaining: 4998

Devices 5 endpoints · /v3/devices

GET /v3/devices/{id}
Retrieve a device

Returns a single DeviceRecord including firmware revision, last-seen timestamp, and the rolling 24h connection-health histogram. Devices that have been offline for more than 72h emit status: "stale".

Parameter
Type
Description
id *
path
Device identifier, e.g. dev_TX-441.
expand
string[]
Comma-separated. Supports streams, alerts.
cURL
Headers
Response
Request
curl "https://api.ferrolink.io/v3/devices/dev_TX-441?expand=streams" \
  -H "Authorization: Bearer $FL_KEY"
200 OK
{
  "id": "dev_TX-441",
  "model": "FerroNode-T4",
  "firmware": "2.11.4",
  "status": "online",
  "site": "plant-04/bay-B",
  "last_seen": "2024-11-22T09:13:58Z",
  "streams": ["strm_8H2kQa"]
}
POST /v3/devices
Register a device

Onboards a new physical FerroNode gateway. The response contains a one-time enrollment_token that must be flashed into the device within 30 minutes via the on-prem provisioning utility.

Field
Type
Description
serial *
string
Hardware serial printed on the gateway label.
site
string
Free-form location label, max 64 chars.
tags
string[]
Up to 16 organizational tags.
cURL
Headers
Response
Request
curl -X POST "https://api.ferrolink.io/v3/devices" \
  -H "Authorization: Bearer $FL_KEY" \
  -H "Content-Type: application/json" \
  -d '{"serial":"FN-T4-00081A2","site":"plant-04/bay-B"}'
201 Created
{
  "id": "dev_LQ8m1k",
  "enrollment_token": "flk_enr_zx9...",
  "expires_at": "2024-11-22T09:44:00Z"
}

Alerts 4 endpoints · /v3/alerts

GET /v3/alerts
List active alerts

Returns currently-firing alerts ordered by severity then triggered_at descending. Alerts that have resolved within the last 30 minutes are included with resolved: true.

Parameter
Type
Description
severity
enum
info, warn, critical.
device_id
string
Filter to alerts originating from a specific device.
cURL
Headers
Response
Request
curl "https://api.ferrolink.io/v3/alerts?severity=critical" \
  -H "Authorization: Bearer $FL_KEY"
200 OK
{
  "data": [{
    "id": "alr_77Hk2",
    "severity": "critical",
    "device_id": "dev_TX-441",
    "metric": "bearing.temp_c",
    "value": 94.2,
    "threshold": 85,
    "triggered_at": "2024-11-22T09:11:02Z"
  }]
}
