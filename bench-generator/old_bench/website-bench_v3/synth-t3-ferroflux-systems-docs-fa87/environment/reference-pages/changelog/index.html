Changelog

Release history for the FerroFlux SDK, spanning 18 months from initial public preview through the current stable line. Versions follow SemVer 2.0.0.

v1.4.2 May 14, 2024
Stabilize WebSocket reconnect under high jitter.
Fixed
  • Exponential backoff in StreamClient.reconnect() no longer overflows after 30+ failures.
  • Race in sensors.list() when polling during firmware refresh.
  • Memory leak in zero-copy buffer pool on Linux 6.x kernels.
v1.4.0 April 02, 2024
Adaptive sampling and per-channel gain profiles.
Added
  • New pipeline.adaptive block in ferroflux.yaml for dynamic sample rates.
  • POST /v1/calibration/profiles endpoint for per-channel gain.
  • Prometheus-compatible /metrics exporter on diagnostic port.
Fixed
  • SPI driver returned stale magnetometer reads after suspend/resume.
  • YAML parser now reports line numbers on duplicate keys.
v1.3.1 February 19, 2024
Hotfix for USB-HID detection on Ubuntu 22.04 LTS.
Fixed
  • udev rule template now references the correct vendor ID range.
  • ferroflux doctor no longer hangs when no device is attached.
v1.2.0 December 06, 2023
Authentication overhaul with API key rotation.
Added
  • auth.keys.rotate() SDK method with grace-period overlap.
  • Scoped tokens: read:sensors, write:calibration, admin:streams.
  • Audit log streamed over wss://<host>/v1/audit.
Fixed
  • Clock skew tolerance increased from 30s to 120s for HMAC signatures.
v1.1.0 October 11, 2023
Multi-sensor fusion preview and TypeScript types.
Added
  • Experimental fusion.kalman() combiner for 2–8 sensors.
  • First-class TypeScript declarations shipped in the SDK package.
  • CLI command ferroflux stream tail for live debugging.
v1.0.0 August 22, 2023
FerroFlux 1.0 — Production-ready release.
Added
  • Stable REST API under /v1/ with versioning guarantee.
  • Backpressure-aware WebSocket transport with frame batching.
  • Official Docker image ferroflux/sdk:1.0.0.
  • Long-Term Support commitment of 18 months for the 1.x line.
Fixed
  • Stabilized buffer-pool sizing under sustained 10 kHz sampling.
  • Resolved 47 issues carried over from the 0.x preview series.
Breaking
  • Removed the deprecated /sensor/poll REST route.
  • Config key buffer_size renamed to pipeline.bufferSize.
  • Callbacks replaced with async iterators on StreamClient.
Migration note — upgrading from 0.x

Run npx ferroflux-migrate@1 in your project root to rewrite legacy ferroflux.yaml files and update SDK call sites. Allow approximately 15 minutes per service; the codemod is idempotent and prints a diff before writing.

v0.6.0 May 30, 2023
Release-candidate cleanup ahead of 1.0.
Added
  • Schema validation for ferroflux.yaml at SDK boot time.
  • GET /v1/streams/:id/health diagnostic endpoint.
Breaking
  • WebSocket frames are now CBOR-encoded by default; JSON is opt-in.
  • Minimum supported Node version raised to 18.17.0.
v0.3.0 February 14, 2023
Public preview: REST endpoints and basic streaming.
Added
  • Initial REST surface for /sensors and /streams.
  • Sample-rate configuration up to 4 kHz on supported hardware.
v0.1.0 November 08, 2022
First internal alpha — magnetic sensor I/O proof-of-concept.
Added
  • USB-HID transport driver for FerroFlux-compatible probes.
  • Minimal Node.js binding exposing read() and close().
  • Initial documentation site skeleton.
