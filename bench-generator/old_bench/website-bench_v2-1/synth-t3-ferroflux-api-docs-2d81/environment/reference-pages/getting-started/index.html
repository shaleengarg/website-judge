Getting Started

Begin streaming real-time magnetic-field telemetry in under five minutes.

⚠ Rate limits on the free tier

Free-tier API keys are capped at 100 requests/minute across all endpoints. Bursts above this threshold return 429 Too Many Requests with a Retry-After header. Upgrade to Pro for 10,000 rpm.

1

Install the FerroFlux SDK

The official SDK is distributed via npm and supports Node 18+, Deno, and modern browsers. It wraps the raw REST API with typed methods and automatic retry on transient 5xx errors.

Terminal
$ npm install @ferroflux/sdk
+ @ferroflux/sdk@2.4.1
added 1 package in 1.4s
Verify installation
$ node -e "console.log(require('@ferroflux/sdk').VERSION)"
2.4.1
2

Authenticate with your API key

Generate an API key from the Developer Portal under Settings → API Keys. Keys are prefixed with ff_live_ for production and ff_test_ for sandbox traffic. Always pass the key in the Authorization header — never in query strings.

Request
$ curl https://api.ferroflux.io/v2/auth/whoami \
    -H "Authorization: Bearer ff_live_8f3a2b9c1e7d4a6f"
Response — 200 OK
{
  "account_id": "acc_19hQz0xK",
  "tier": "free",
  "rate_limit": 100,
  "scopes": ["sensors:read", "streams:subscribe"],
  "expires_at": "2026-04-12T00:00:00Z"
}
3

Send your first sensor-poll request

Each FerroFlux sensor is addressed by a stable identifier. The /v2/sensors/{id}/reading endpoint returns the most recent triaxial magnetic-field vector along with calibration metadata. Readings are emitted at 200 Hz; polling above 5 Hz should use a stream instead.

Request
$ curl https://api.ferroflux.io/v2/sensors/snr_4f7e2a/reading \
    -H "Authorization: Bearer ff_live_8f3a2b9c1e7d4a6f" \
    -H "Accept: application/json"
Response — 200 OK
{
  "sensor_id": "snr_4f7e2a",
  "timestamp": "2025-04-12T17:42:09.318Z",
  "field": {
    "x_uT":  23.471,
    "y_uT": -11.208,
    "z_uT":  42.916,
    "magnitude_uT": 49.812
  },
  "temperature_c": 21.4,
  "calibration": {
    "profile": "indoor_v3",
    "drift_ppm": 12
  },
  "stale": false
}
