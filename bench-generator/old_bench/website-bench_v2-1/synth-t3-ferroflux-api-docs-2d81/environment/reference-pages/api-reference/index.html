API Reference

Complete reference for the FerroFlux REST API. All endpoints are rooted at https://api.ferroflux.io/v2 and require a bearer token in the Authorization header.

Sensors 5 endpoints
GET /sensors List all registered magnetic-field sensors

Description

Returns a paginated list of sensors visible to the current API key. Use query filters to narrow the result set by region, calibration state, or firmware version.

Query Parameters

NameTypeRequiredDescription
limitintegeroptionalMax items per page (1–200, default 50)
cursorstringoptionalOpaque pagination cursor from prior response
regionenumoptionalOne of us-east, eu-west, ap-south
calibratedbooleanoptionalFilter by calibration completion state
Request
curl https://api.ferroflux.io/v2/sensors \
  -H "Authorization: Bearer $FF_KEY" \
  -G --data-urlencode "limit=25" \
     --data-urlencode "region=eu-west"
Response 200
{
  "data": [
    {
      "id": "snr_8f2c1a",
      "region": "eu-west",
      "calibrated": true,
      "firmware": "3.1.0"
    }
  ],
  "cursor": "eyJvIjoyNX0="
}
GET /sensors/{id} Retrieve a single sensor by ID

Description

Returns full metadata for the specified sensor including hardware revision, last sync time, and active calibration profile.

Path Parameters

NameTypeRequiredDescription
idstringrequiredSensor identifier (snr_*)
Request
curl https://api.ferroflux.io/v2/sensors/snr_8f2c1a \
  -H "Authorization: Bearer $FF_KEY"
POST /sensors Register a new sensor on the network

Body Parameters

NameTypeRequiredDescription
serialstringrequiredHardware serial number
regionenumrequiredTarget deployment region
labelstringoptionalHuman-readable name (≤64 chars)
profile_idstringoptionalCalibrationProfile to bind on register
Request
curl -X POST https://api.ferroflux.io/v2/sensors \
  -H "Authorization: Bearer $FF_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "serial": "FF-22A-91041",
    "region": "us-east",
    "label": "Lab Bench North"
  }'
PUT /sensors/{id}/calibrate Trigger a calibration cycle

Description

Initiates an asynchronous calibration. The response includes a job handle that can be polled via the same endpoint with GET.

Body Parameters

NameTypeRequiredDescription
profile_idstringrequiredCalibrationProfile reference
samplesintegeroptionalSample count override (default 1024)
Response 202
{
  "job_id": "job_a1b2c3",
  "status": "queued",
  "eta_seconds": 42
}
DELETE /sensors/{id} Permanently decommission a sensor

Marks the sensor as decommissioned. Historical readings are retained for 90 days, after which they are purged.

Request
curl -X DELETE \
  https://api.ferroflux.io/v2/sensors/snr_8f2c1a \
  -H "Authorization: Bearer $FF_KEY"
Streams 5 endpoints
GET /streams List active stream sessions

Query Parameters

NameTypeRequiredDescription
sensor_idstringoptionalFilter by source sensor
stateenumoptionalopen, paused, or closed
Response 200
{
  "streams": [
    {
      "id": "str_77ee01",
      "sensor_id": "snr_8f2c1a",
      "state": "open",
      "hz": 200
    }
  ]
}
POST /streams Open a new real-time stream session

Body Parameters

NameTypeRequiredDescription
sensor_idstringrequiredSource sensor for the stream
hzintegerrequiredSample rate (1–1000 Hz)
transportenumoptionalws (default) or sse
compressionenumoptionalnone, gzip, or zstd
Request
curl -X POST https://api.ferroflux.io/v2/streams \
  -H "Authorization: Bearer $FF_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "sensor_id": "snr_8f2c1a",
    "hz": 200,
    "transport": "ws"
  }'
Response 201
{
  "id": "str_77ee01",
  "ws_url": "wss://stream.ferroflux.io/v2/str_77ee01"
}
GET /streams/{id} Retrieve stream session metadata

Returns the current configuration, frame counters, and last error (if any) for the specified stream session.

Path Parameters

NameTypeRequiredDescription
idstringrequiredStream session id (str_*)
Request
curl https://api.ferroflux.io/v2/streams/str_77ee01 \
  -H "Authorization: Bearer $FF_KEY"
PUT /streams/{id}/pause Pause an open stream without closing it

Pausing preserves the session token and resumes within 60 seconds. After 60 seconds the stream auto-closes and must be re-opened.

Request
curl -X PUT \
  https://api.ferroflux.io/v2/streams/str_77ee01/pause \
  -H "Authorization: Bearer $FF_KEY"
DELETE /streams/{id} Close a stream session

Idempotent. Closing an already-closed stream returns 204 No Content.

Response 204
// No content
Webhooks 4 endpoints
GET /webhooks List configured webhook subscriptions

Query Parameters

NameTypeRequiredDescription
eventstringoptionalFilter by event type pattern
activebooleanoptionalOnly enabled subscriptions
Response 200
{
  "webhooks": [
    {
      "id": "whk_5e80aa",
      "url": "https://hooks.acme.io/ff",
      "events": ["sensor.alert"],
      "active": true
    }
  ]
}
POST /webhooks Create a new webhook subscription

Body Parameters

NameTypeRequiredDescription
urlstringrequiredHTTPS endpoint to receive payloads
eventsstring[]requiredArray of event patterns to subscribe
secretstringoptionalHMAC signing secret (32–128 bytes)
Request
curl -X POST https://api.ferroflux.io/v2/webhooks \
  -H "Authorization: Bearer $FF_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://hooks.acme.io/ff",
    "events": ["sensor.alert","stream.error"],
    "secret": "whsec_9d2f..."
  }'
PUT /webhooks/{id} Update an existing subscription

Partial updates supported. Only fields present in the request body are modified; omitted fields preserve their existing values.

Path Parameters

NameTypeRequiredDescription
idstringrequiredWebhook id (whk_*)
Request
curl -X PUT \
  https://api.ferroflux.io/v2/webhooks/whk_5e80aa \
  -H "Authorization: Bearer $FF_KEY" \
  -H "Content-Type: application/json" \
  -d '{ "active": false }'
DELETE /webhooks/{id} Remove a webhook subscription

Removes the subscription immediately. Pending deliveries already in the dispatch queue may still fire for up to 30 seconds.

Response 204
// No content
